#Sistema de Gestión de Recibos y Cobranzas - Gestion_Agua_SacSistema web para la administración de consumo de agua, facturación mensual y control de estados de medidores. Desarrollado como proyecto final para el curso de Java Web en SENATI, utilizando una arquitectura robusta y persistencia de datos profesional.Descripción del NegocioNombre: Sistema de Facturación "Gestion_Agua_Sac"Giro: Empresa de saneamiento y distribución de agua potable.Tamaño: Pequeña/Mediana empresa con operaciones de recaudación local.Contexto: En muchas zonas, el control del consumo de agua y el estado de los medidores se lleva de forma manual. Este sistema digitaliza el proceso desde el registro del cliente hasta la emisión del recibo mensual.Justificación: Se requiere reemplazar el registro físico por una plataforma web que permita identificar rápidamente quién "YA PAGO" y quién tiene deudas pendientes, además de monitorear medidores malogrados.Identificación del Problema y SoluciónProblema: La dificultad para calcular montos totales basados en el consumo y la pérdida de control sobre los recibos vencidos, sumado a la falta de un inventario actualizado del estado de los medidores de los clientes.Solución Tecnológica: Desarrollo de una aplicación web con Java Spring Boot 3 y MySQL que automatiza la generación de recibos, gestiona las tarifas según el tipo de uso y permite actualizar el estado del medidor en tiempo real.Requerimientos FuncionalesCódigoDescripciónRF01El sistema debe registrar clientes con DNI, dirección, celular y estado del medidor.RF02El sistema debe permitir la creación de recibos mensuales asociados a un cliente.RF03Control de Pagos: Actualizar el campo ya_pago de 'NO' a 'SI' al confirmar el cobro.RF04El sistema debe calcular el monto_total según la cantidad_agua_usada.RF05Listado de clientes con filtros por estado de medidor (Bueno, Malogrado, Sin Medidor).Requerimientos No FuncionalesRNF01 (Rendimiento): Las consultas de recibos por cliente deben procesarse en menos de 2 segundos.RNF02 (Usabilidad): Interfaz intuitiva diseñada en Figma para que el personal de cobranza registre pagos rápidamente.RNF03 (Seguridad): Acceso protegido al sistema mediante credenciales de usuario (Spring Security).Pila Tecnológica (Stack)Trello: Gestión de tareas y entregables del proyecto.Draw.io: Modelado de diagramas Entidad-Relación de la BD Gestion_Agua_Sac.MySQL Workbench: Administración de las tablas clientes, tarifas y recibos.IntelliJ IDEA: Desarrollo del backend (Java 17 / Spring Boot 3).XAMPP: Servidor local para ejecutar MySQL y el contenedor Tomcat.Frontend: HTML5, CSS3, JavaScript y Thymeleaf para la renderización de datos.Estructura del Proyecto en JavaPlaintextGestion_Agua_Sac/
